Programación estructurada

Se le conocerá como programación estructurada a la forma en como un código de cualquier lenguaje estará ordenado en una estructura lógica que lo hace fácil de entender y modificar. Esta programación está basada en el uso de estructuras básicas como secuencia, selección o iteración, universales en cualquier tipo de algorritmo.

Con esto ya es posible darse una idea de que trata este tipo de programación. Básicamente que el código que se encuentre desarrollando uno sea limpio y sencillo. Esto tendrá varios beneficios, generalmente para el programador ya que podrá recordar su código y continuarlo sin problemas, y también beneficiará a la empresa para la que trabaje este programador beneficiando a su vez a otras personas que trabajen con el programador. Si un programador desea compartir su código en una plataforma coo GitHub, también será de gran ayuda.


La programación estructurada tiene como una de sus principales características la modularidad, que significa la división del código de un programa en varios módulos que serán archivos de códigos separados de un archivo de código principal. Esto, haciendo que un archivo principal tenga varios módulos, y cada uno de estos módulos es llamado por el código principal cuando se lo necesita para realizar una función en específico. Si bien un código puede tener todas sus funciones en un solo archivo, en programas grandes siempre se trabaja con módulos debido a que un solo código sería demasiado complejo y extenso para editar, haciendo más sencillo un código en específico.


Historia de la progarmación estructurada

La programación nació con los avances en la aún entonces primitiva computación alrededor de la década de los 60, donde se utilizaban con frecuencia objetos como las tarjetas perforadas, los procesos para codificar y ejecutar un programa eran demasiados lentos, por lo que pronto se observó la necesidad de ordenar los códigos que se iban creando, es por eso que en 1968 Edsger Dijstra publica un artículo donde mostraba varios problemas que tenía la programación en ese entonces, esto se tomó en cuenta y dos años después, en 1970, se desarrolló el lenguaje Pascal, el primero en aplicar el concepto de programación estructurada. Hoy, la mayoría de lenguajes se basan mediante este tipo de programación, siendo Python el más conocido. En la imagen, una tarjeta perforada.


Tipos de programación estructurada

Existen dos tipos de programación estructuarada, estos son:

  • Programación procedural: Según este tipo de programación las funciones son llamadas con unos párametros y órdenes en específico para realizar una acción, que se conocerán también como proceduras. Las funciones en este caso tendrán pocas expresiones, de esta forma cumplirán un solo objetivo y la función se llamará únicamente cuando se necesita de verdad.

  • Programación orientada a objetos: Esta programación, que es mucho más conocida, define que un programa se compone de objetos de los cuales vendrán las acciones de un programa. Cada uno de estos objetos tendrá características y atributos propias, que serán muy útiles cada que sea necesario. Debido a su dinámica, que permite establecer como objetos la mayoría de funciones que puede tener un software, es el tipo de programación estructurada más usado. Puedes leer más sobre la programación POO en el siguiente artículo, dedicado completamente a este tema de programación.

  • Programación basada en modelos: No tan conocida como las dos anteriores, pero no menos importante, este tipo de programación se basa en la abstracción de la lógica de la aplicación y los modelos de datos creando un código fuenta a partir de modelos validados. Tiene como principal objetivo permitir la mayor eficiencia de desarrollo y la subsecuente reutilización del software a través de sistemas con códigos más abstractos. Una representación abstracta captura captura los concepto centrales, relaciones y patrones de comportamiento de un sistema.

    • Porqué usar programación estrucurada

    Además de que hace los códigos más comprensibles para el público, la programación estructurada puede destacar varias ventajas, todas derivadas de la simplicidad del código, porque un código simple puede ser tomado como una plantilla y ser reutilizado para distintos fines, así mismo como los módulos (en programación procedural) o clases (en programación orientada a objetos) que se pueden aplicar en distintos programas al realizar cada uno de estos una sola función (por ejemplo: supongamos que alguien creó un módulo para saber si una persona es mayor de edad, debido a que hay muchas cosas que solicitan esto, este módulo podría ser usado en distintos programas que requieran de esto).


    Además, el organizar un hace también que los tiempos de desarrollo de un programa sean mucho más rápidos y eficientes. Estas son razones suficientes para que este tipo de programación sea el más usado de todos en el desarrollo de software. Respecto a las desventajas, se podría decir de una menor eficiencia y de una sobrecarga del código de estructuras, pero esto se da solamente en ciertos casos. Así, que, la programación estructurada es la mejor opción para el desarrollo de software, quizá la que estás aprendiendo y aún no te des cuenta.
